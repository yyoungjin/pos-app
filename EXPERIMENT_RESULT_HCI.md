# POS UI 비교 실험 결과 정리 (HCI 관점)

## 1) 실험 목적
- 동일한 과업(`taskId`)에 대해 `엑셀뷰`와 `그리드뷰`의 수행 성능 차이를 비교한다.
- 비교 축은 다음 2가지다.
  - **효율성(Efficiency)**: 과업 완료 시간(`responseTimeMs`)
  - **정확성/오류(Effectiveness, Error)**: 오클릭 횟수(`misclickCount`)

## 2) 데이터 개요
- 참가자 수: **12명**
- 과업 수: **6개(T01~T06)**
- 총 비교 시도: **72회** (12명 x 6과업)
- 데이터 소스: `src/data/adminMockUserResponses.json`

## 3) 핵심 결과 (요약)
- 전체 평균에서 `엑셀 - 그리드` 시간 차는 **+132ms**
  - 즉, 평균적으로 **그리드뷰가 더 빠름**
- 빠른 케이스 분포
  - **그리드뷰 더 빠름: 47회**
  - **엑셀뷰 더 빠름: 25회**
- 오클릭 평균
  - **엑셀뷰: 2.33회**
  - **그리드뷰: 3.15회**
  - 즉, 그리드뷰가 빠르지만 오클릭은 더 많은 경향

## 4) 태스크별 결과

해석:
- 대부분 과업에서 그리드뷰가 더 빠르며(+), 특히 `T03`, `T06`에서 차이가 큼.
- `T05`에서는 엑셀뷰가 더 빠르게 나와, **뷰 우위가 과업 특성에 따라 바뀔 수 있음**을 시사.
- 전 과업에서 오클릭은 그리드뷰가 더 높아, **속도-정확성 트레이드오프**가 나타남.

## 5) HCI 관점 해석

### 5-1. Efficiency vs. Effectiveness 트레이드오프
- 그리드뷰는 탐색/스캔이 빠른 구조여서 반응 시간이 짧아질 가능성이 높다.
- 반면 클릭 정확도는 상대적으로 떨어져 오클릭이 증가했다.
- 이는 전형적인 **속도-정확성 상충관계(speed-accuracy trade-off)**로 해석 가능하다.

### 5-2. 인지 부하(Cognitive Load)
- 엑셀뷰는 규칙적 정렬과 표 구조로 인해 정확한 매칭에는 유리하지만, 시각적 검색 시간이 길어질 수 있다.
- 그리드뷰는 시각적으로 빠르게 눈에 들어오지만, 근접한 카드/영역 간 혼동으로 오클릭이 증가할 수 있다.

### 5-3. 과업 의존성(Task-UI Fit)
- `T05`처럼 특정 과업은 엑셀뷰가 더 잘 맞을 수 있다.
- 즉 “어떤 UI가 항상 우수하다”보다는 **과업 특성에 따라 적합 UI가 다름**이 중요하다.

## 6) 아쉬운 점
- 현재는 요약 지표 중심이라, 클릭 경로(어떤 항목을 어떻게 잘못 눌렀는지) 분석이 제한적이다.
- 학습효과를 관측하려면 동일 과업 반복 설계가 필요한데, 현재는 과업당 1회라 추세 해석이 약하다.
- 오클릭의 유형(인접 오류/카테고리 오류/테이블 오류) 구분이 없어 인터랙션 문제의 원인 추적이 어렵다.

## 7) 한계
- 사용자 숙련도, 피로도, 화면 크기/해상도, 입력 장치(마우스/트랙패드) 같은 교란 요인을 통제하지 못했다.
- 실험 조건(시간 압박, 튜토리얼 강도, 시야 이동량)도 실제 환경과 다를 수 있다.

## 8) 다음 실험 제안
- **반복 측정 설계**: 각 과업을 최소 2~3회 반복해 학습효과를 분리 추정
- **오류 유형 라벨링**: 오클릭을 원인별로 분류해 UI 개선 포인트 도출
- **혼합 지표 사용**: 시간 + 오클릭 + 주관적 만족도(SUS/NASA-TLX) 동시 측정
- **과업별 UI 추천**: 단일 승자 UI 대신 과업 유형에 따른 적응형 UI 전략 검토

## 9) 결론
- 현재 결과는 “**그리드뷰는 더 빠르고, 엑셀뷰는 더 정확한 경향**”을 보여준다.
- 이는 HCI에서 중요한 트레이드오프이며, 실제 적용 시에는 **속도 우선 맥락**과 **정확도 우선 맥락**을 구분한 설계가 필요하다.
